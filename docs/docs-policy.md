# Docs Policy

## Principle

- Put shared rules, background, terminology, and workflows in the root `docs/` directory
- Put any content referenced by multiple skills in `docs/`
- Keep `agent.md` limited to summaries and links

## Writing Guideline

- Keep each document focused on a single responsibility
- Use filenames that clearly describe their role
- Avoid duplicated explanations and prefer links when the same topic appears in multiple places
- Write all documentation in English
- Write code comments in English

## Update Rule

- Update `docs/` first when changing shared rules
- Keep links from `agent.md` valid
- Replace duplicated explanations in skills with links back to `docs/`
