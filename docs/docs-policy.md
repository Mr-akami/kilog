# Docs Policy

## Principle

- 共通ルール、背景、用語、フローはルートの`docs/`に置く
- 複数のSkillから参照される内容は、必ず`docs/`に置く
- `agent.md`には要約とリンクだけを書く

## Writing Guideline

- 文書は単一責務にする
- ファイル名は役割が分かるものにする
- 重複説明を避け、必要なら参照リンクに寄せる

## Update Rule

- 共通ルールを変えるときは、先に`docs/`を更新する
- `agent.md`のリンク切れを防ぐ
- Skill側に重複説明がある場合は、`docs/`参照へ寄せる
